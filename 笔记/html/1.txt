-> 结构标记：
	-> 作用：
		->经常使用<div> 元素设计页面的大致布局,如页头、导航栏、主要内容部分和页脚等
		-> 布局复杂时,会出现大量的<div>元素
		-> 元素相互形成嵌套,页面会难以处理和护理

-> HTML提供了结构标记,专门用于标识常见的结构
	-> 如页头、页脚等
	-> 可以很方便的实现各个部分的划分
	-> 使得文档结构更清晰明确,代码更容易阅读
	
-> 结构标签：
	-> <header></header>  //这个是页头
		-> 用于定义文档的页眉
		-> 通常是一些介绍信息、导航信息、站点标题或logo图片等
		-> 可以在页面上出现多次、可以作为任何部分定义
	-> <nav></nav>    //这个就是导航
		-> 用来定义页面的导航链接部分(主导航)
		-> 用于包含表示链接的其他元素
	-> <section></section>		//这个就是主信息
		-> 用于定义文档中的节
		-> 表示文档中的一个具体的组成部分
			-> 通常用于为页面上的内容分块、比如定义章节、页眉、页脚或文档中的其他部分
			-> 可以使用<header>元素为内容添加标题
	-> <aside></aside>       //这个就是边栏
		-> 常用于定义页面上的一些额外组成部分
		-> 如：广告栏、侧边栏和相关引用信息等
	-> <footer></footer>	 //这个就是页脚
		-> 常用于定义某区域的脚注信息
		-> 常用来定义文档的页脚、以包含友情链接、版权信息和作者信息等
		-> 一个页面中也可以包含多个<footer>元素
	-> <article></atricle>	 //article(文章) 
		-> 该标签经常用于定义独立于文档的其他部分的内容
			-> 比如论坛的一个帖子、一篇报纸文章、某个博客条目或者用户评论等

-> 表单			
	-> 表单的作用：
		-> 用于显示、收集信息、并提交到服务器
		-> 表单有两个基本部分
			-> 实现数据交互的可见界面元素、比如文本框或按钮
			-> 提交后的表单处理
	-> 界面元素：
		-> 使用<from>元素创建表单
		-> 在<from>元素中添加其他表单可以包含的控件元素
	-> 定义表单：
		-> 使用成对的<from></from>标记
		-> 主要属性：
			-> action：定义表单被提价时发生的动作,通常包含服务器方脚本的url
			-> method：指出表单数据的提交方式,取值为get或者post
			-> enctype:表单数据进行编码的方式
			-> name:表单的名称
	-> text的maxlangth属性可以设置文本框的最大字节数
	-> disabled：设置文本不能使用
	-> readonly：只读
	-> required：如果为文本框设置该属性,则当文本框为空的时候不能提交 不兼容低版本 h5新属性
	-> name属性在radio或者CheckBox中只有name属性一致的时候才能实现单选
		-> name属性其实就是分组
		-> 单选框必须设置value值才能提交上去  name属性一致提交value的值上去
	-> select
		-> multiple：允许多选 仅适合多行菜单   selected：定义默认选中
	-> 多行文本框(文本域)：
		-> <textarea>
			-> name属性：
			-> cols：设置列数
			-> rows：设置行数
		-> 写在textarea标签中间的值会被当做是默认值出现在你的文本框中
	-> label标记：
		-> 可以与我们的表单元素进行绑定,比如和复选框绑定,点击label里面的内容可以勾选复选框
		-> 想要绑定必须在表单元素中添加id,然后在label中用for属性绑定id,for的值填写id的值
	-> reset：重置按钮
	-> hidden：隐藏域
		-> 放用户看不见的内容
	-> 表单提交：
		-> 属性：
			-> name：给表单取名字
			-> method：表单提交方式
				-> get：默认提交方式,将信息附加在地址栏提交
				-> post：将表单数据打包,把包提交,安全
			-> action：一般是用来服务器端用来处理表单数据的文件路径
				-> 也可以填写mailto：填写邮箱,通常包含服务器方脚本的url
			-> enctype：定义表单数据的编码格式
			
-> 控件分组标记：
	-> <fieldset>元素：为控件分组  (这个相当于是GroupBox)
	-> <legend>元素：为分组指定一个标题 (这个就是GroupBox的名称)
		-> 把需要分组的内容写在fieldset里面 legend下面
		
-> 其他常用标记
	-> 浮动框架<iframe>元素
		-> 可以在一个浏览器窗口中显示多个页面文档,就是在当前页面显示其他页面内容
		-> 在页面上使用<iframe>元素
		-> 设置<iframe>元素的src属性,指向其他页面的URL
		-> frameborder属性：只能取值0或者1
			-> 0：无边框
			-> 1：有边框
		-> name属性：
			-> 我们可以使用a标签的target属性填写iframe的name属性的值,可以将制定页面显示在
				iframe框架中
		-> scrolling属性：可以设置是否显示滚动条
			-> yes：显示	
			-> no：不显示
		-> 在iframe标签中写内容,当iframe连接到的内容无法正常显示的话,则显示标签中的内容
	-> 摘要与细节：
		-> 网页中经常需要将部分信息进行展开或者伸缩
		-> <details>元素用于定义细节
		-> <summary>元素用来包含<details>元素的标题
		-> 目前只有Chrome支持
	-> 度量元素<meter>
		-> <meter>元素用于定义度量衡
			-> 比如用于表示投票人数比例。磁盘的使用量或者统计比例等
			-> 常用于静态比例的显示(已知最大值和最小值)
		-> 语法：
			-> <meter>文本</meter>
		-> 主要属性：
			-> min：范围的最小值,默认为0
			-> max：范围的最大值,默认为1  (可以为任何数字)
			-> value：度量值,默认值为0  (当前值)
	-> 时间元素(time)：
		-> <time>元素用来定义公立的时间(24小时制)或日期
			-> 并不能为页面带来页面显示效果上的变化
			-> 常用于对网页添加与时间相关的附加信息
		-> 语法：
			-> <time>文本</time>
		-> 主要属性：
			-> datetime：规定日期/时间,日期与时间之间用"T"文字分割
				2018-3-17T11:24
	-> 高亮文本显示<mark>元素
		-> 定义页面中带有记号的文本
			-> 常用于需要突出显示的文本
			-> 被<mark>元素包围起来的文本会显示额外的背景色
